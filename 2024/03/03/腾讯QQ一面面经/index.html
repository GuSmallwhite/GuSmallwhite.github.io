为什么一致性哈希要虚拟节点？

(我是弱智，这么重要的东西，我也知道他很重要，但是就是没看)

因为一致性哈希并不能保证节点在环上的均匀分布

要想解决节点能在哈希环上分配不均匀的问题，就是要有大量的节点，节点数越多，哈希环上的节点分布的就越均匀。

但问题是，实际中我们没有那么多节点。所以这个时候我们就加入虚拟节点，也就是对一个真实节点做多个副本。

节点数量多了后，节点在哈希环上的分布就相对均匀了。这时候，如果有访问请求寻址到「A-01」这个虚拟节点，接着再通过「A-01」虚拟节点找到真实节点 A，这样请求就能访问到真实节点 A 了。

Netty为什么要异步？说说具体场景不异步会怎么样

  1. 资源利用率低：在同步模型中，当线程被阻塞时，会导致系统的CPU和内存资源无法充分利用，因为线程被阻塞时处于空闲状态，无法执行其他任务。这会影响系统的响应速度和吞吐量。
  2. 易引发死锁：在高并发环境下，同步IO模型容易出现死锁情况。例如，如果一个线程在等待某个IO操作完成时被阻塞，而其他线程又在等待该线程释放资源，就可能导致死锁。
  3. 不利于扩展性：同步IO模型不利于系统的扩展，因为每个连接都需要一个独立的线程来处理，当连接数增加时，线程数量也会增加，系统的性能和稳定性就会受到限制。

redis中的+1-1操作是立即执行还是有延迟的？

GPT说：

在 Redis 中，对于 +1 或 -1 这样的操作是立即执行的，不会有延迟。这是因为 Redis 是单线程的，所有的操作都是依次执行的，不会出现并发竞争的情况。当执行一个 +1 或 -1 的操作时，Redis 会立即对对应的键进行增加或减少操作，并且返回操作后的结果。
